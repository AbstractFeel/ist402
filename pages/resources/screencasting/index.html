Checklist

Before recording always make sure...

  • You've got the links you want to view open in tabs in a browser window
  • You've got headphones / some kind of microphone to record your voice
    • You can use internal microphones but the quality on them is pretty bad so try to use a headset / earbuds if you have them
  • You've authorized whatever it is to record your voice and test that audio is working before recording!
    • I fell victim to this many times early on as did a few students in previous semesters

Next you'll need to pick a screen casting tool to do the recording. I recommend one of the following.

Zoom (Free)

Link : https://cmm.psu.edu/zoom/

Penn state labs come with zoom installed. You can also install this on your personal device and should be able to do recordings from it. I recommend finding a quite place that you can bring headphones since you'll have to do voice over describing what's on screen and explaining it to others.

Screencastify (Free, ad based with paid option, $29/year, via Chrome)

Link : https://chrome.google.com/webstore/detail/screencastify-screen-vide/mmeijimgabbpbgpdklnllpncmdofkcpn?hl=en

I just found this recently, it's a browser plugin for Chrome. If you're O.K. with the permissions that it requires , it's the fastest way of generating a screencast and get it up to YouTube. I use this when screencasting from my Ubuntu/PC.

Snagit (Paid $30, Mac/PC)

Link :  https://www.techsmith.com/snagit-pricing.html

I use Snagit personally for all of my screen casting that I do. It is $30 but comes with some basic ways of doing quick editing and image mock up as well. It's not required that you use Snagit by any means, just saying what I've had positive experiences with previously. I use this when screencasting from my Mac.

Others

If you find others that you like feel free to share them with the class. The purpose of this class is not to get into high end video editing so beware that it is expected you'll generate videos each week. The content of the video is what matters, not fancy intros or well cut jumps or anything like that. Being able to fluidly communicate about a technical concept and teach others about how to use it and what it is.
