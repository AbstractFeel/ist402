Basic example / Getting started

  1. Create a GitHub account
  2. Fork this repo by going to this link and clicking "Fork" in the top right
  3. Read along in the document and complete the tasks asked of you by navigating to the README.md file and hitting edit.
  4. We'll work through this in class and I'll mix between showing things interactively done between specific people as well as roaming around to answer questions
  5. After you make the updates requested in the document, issue a "Pull Request" back against the source (my repo)
  6. Fill out the details indicating what you changed and we'll talk through it in class how this is the building block for all web based development collaborations

Let's start contributing to an open source project!

Let's start contributing to our first open source project! I've left a few issues in my own projects so that we can see the full project development life cycle at play here. Don't worry, these are minor documentation and spelling issues which while seeming minor, are a HUGE deal to developers and project managers as far as community involvement.

EVERYONE (me included) start out getting into a development community by updating documentation, making sure acronyms made sense and fixing links and typos. It's the best way to get into a software community is to fix things around the edges and ensure that you can effectively communicate with people in the project. It also helps prove they are human.

  1. Select one of the issues from this list of issues
  2. Read the description of what needs fixed, ask questions if you need clarification, and comment in the thread indicating that you plan on working on it.
  3. Hit FORK on the repo, fix the issue and submit a pull request.
