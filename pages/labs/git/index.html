Basic example / Getting started

  1. Create a GitHub account
  2. Fork the repo backing this course
  3. Go to files/spring2020.md and edit the file by clicking the pencil
  4. Write markdown that adds the name of your brand and a link to your YouTube channel
  5. Add a commit message saying that you added your link and hit Commit
  6. Click Pull Request  and create a new pull request
  7. In the message for it indicate that you added the link to your channel and submit the PR.
  8. We'll do this in class and look at the graph it forms

Contributing to projects

EVERYONE (me included) start out getting into a development community by updating documentation, making sure acronyms made sense and fixing links and typos. The best way to get into a software community is to fix things around the edges and ensure that you can effectively communicate with people in the project. It also helps prove they are human.

For the lab this week let's build a living glossary of the terms and technologies we're going to cover this semester and then use the standard way of contributing to projects in order to contribute back to the course.

  1. Go to the course repo and see the files/glossary.md file. This is the file that's powering the content in the Additional resources / Glossary portion of the course website
  2. Look through the listed and claim a term that you are going to contribute a definition for.
  3. How to claim a term:
    1. Go to the issue queue for the course
    2. Create a new issue called "Definition for ____" where the blank is the word you are going to define. Once a word is listed you must select a different one
  4. Do some research to come up with a working definition of the word for the course. Write your definition for it in markdown in the file below the heading for the word
  5. Provide 2 to 3 links to additional resources about the defined word
  6. Issue a Pull Request back to the course repository with the name of the topic that you selected as the title of the PR
