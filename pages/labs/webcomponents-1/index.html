How to submit to lab

Similar to last week, we're going to fork and modify something but then we have a specific type of topic prompt for the response:

  1. Fork this repo just like we did last week
  2. Update the details in example-app to relate to you
  3. Copy the example-app code and fill it out differently, demonstrating stacking / time savings
  4. update the CSS code to remove the yellow background and change the border color of the css, impacting all copies (again, scale)
  5. Do your written response where it says "Here's where you should fill in your response "
  6. Pick one of these to frame your writing and video response:
    • Make the business , security , accessibility,  OR design case for using web components, leveraging links to support your case or through demonstration in one of the repos we've stepped through.
    • Code / Developer route : Run through Try LitElement and do a video explaining the value of LitElement, with supporting links.
    • Package manager / tooling pro : Setup NPM / node and create an element using Open-WC, the tooling the pros use https://open-wc.org/guide/  and do a video on that process / making a hello-world element in it.
  7. Record your video and link to it from your fork of the repo next to the writing area
  8. Post in #lab-4-webcomponents  in our slack group
  9. Purchase a domain / account on Reclaim hosting to prepare for next week
